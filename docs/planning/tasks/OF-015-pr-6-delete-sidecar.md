# OF-015 PR 6 — Delete side-car infrastructure + iron-log cleanup + `FallbackRecord` removal

## Goal

With ontogen-ts driving the long-tail emission as of PR 5, delete the now-dead side-car infrastructure and iron-log's consumer-side workarounds in one focused PR. Removes: `ts_sidecar.rs`, the side-car helpers in `src/servers/mod.rs`, the `ONTOGEN_TS_SIDECAR_INNER` env guard, the `FallbackRecord` type and its warning emission paths, iron-log's `.taurignore`, `default-run`, `IRON_LOG_SKIP_SERVER_CODEGEN` env-gate, and `specta-typescript` dependency. Satisfies AC-12, AC-13, AC-14 of [OF-015](./OF-015-productionize-typescript-generation.md).

## Today

After PR 5, the side-car code is dead but still in-tree:
- `src/servers/generators/ts_sidecar.rs` is not called from anywhere.
- `src/servers/mod.rs::sidecar_lib_crate_name` and `sidecar_types_module_path` are unreferenced.
- `src/servers/mod.rs::generate_transport` still checks `ONTOGEN_TS_SIDECAR_INNER` at its top (the guard short-circuits the no-longer-relevant recursion case).
- `src/transport.rs::FallbackRecord` and the related warning-emission paths in `src/servers/generators/ts_client.rs` exist but ontogen-ts never produces them (hard-error model per OF-015 design).
- `examples/iron-log/src-tauri/.taurignore` exists.
- `examples/iron-log/src-tauri/Cargo.toml` carries `default-run = "iron-log"` under `[package]` and `specta-typescript = "=0.0.10"` under `[dependencies]`.
- `examples/iron-log/src-tauri/build.rs` carries the `IRON_LOG_SKIP_SERVER_CODEGEN` env-gate idiom.
- The generated `examples/iron-log/src-tauri/src/bin/__ontogen_ts_export.rs` is no longer regenerated by PR 5's wiring, but any stale copy in the working tree should be deleted.

## Approach

Three commits inside the worktree:

1. **Delete the side-car module and its helpers** in `src/`:
   - Delete `src/servers/generators/ts_sidecar.rs`.
   - Remove `mod ts_sidecar;` (and `pub use`) from `src/servers/generators/mod.rs`.
   - Remove `sidecar_lib_crate_name` and `sidecar_types_module_path` from `src/servers/mod.rs`.
   - Remove the `ONTOGEN_TS_SIDECAR_INNER` env-guard check from the top of `generate_transport`.
   - `cargo build` succeeds; no `cargo run` of any inner build-script binary remains in `gen_servers`'s code path.

2. **Remove `FallbackRecord`** and its warning paths:
   - Delete the `FallbackRecord` type in `src/transport.rs` and any related emission helpers.
   - Remove `FallbackRecord` emission from `src/servers/generators/ts_client.rs`.
   - Remove any tests that exercised the fallback-warning path (or convert them to assert the hard-error model).
   - Note in the cutover changelog/release notes that any consumer grep rule matching the old warning text (e.g., `"ontogen: TS bindings: fallback Record<string, unknown>"`) will no longer match — this is mentioned in PR 8's docs work.

3. **Clean up iron-log's workarounds**:
   - Delete `examples/iron-log/src-tauri/.taurignore`.
   - Remove `default-run = "iron-log"` from `examples/iron-log/src-tauri/Cargo.toml [package]`.
   - Remove `specta-typescript = "=0.0.10"` from `examples/iron-log/src-tauri/Cargo.toml [dependencies]`. (Note: `specta` itself stays — Tauri IPC bridge uses it.)
   - Remove the `IRON_LOG_SKIP_SERVER_CODEGEN` env-gate from `examples/iron-log/src-tauri/build.rs`.
   - Delete any stale `examples/iron-log/src-tauri/src/bin/__ontogen_ts_export.rs` if present in the working tree.
   - Verify `cargo build` in `examples/iron-log/src-tauri/` still succeeds; spot-check `bindings.ts` is unchanged or improved.

Each commit builds clean and `just full-check` passes.
